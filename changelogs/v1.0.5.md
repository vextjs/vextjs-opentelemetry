# v1.0.5

> 发布日期：2026-04-28
> 类型：PATCH

## 概述

本次补丁聚焦 `instrumentation` 在 `deferred export` 场景下的启动摘要误导问题：默认不再打印 `SDK initialized → deferred export ...`，避免 VextJS 用户把启动期的阶段性状态误解为最终未配置或未上报。

## 变更明细

### 启动日志策略调整

- `src/instrumentation.ts`
  - 当 `exportMode === "none"` 时默认静默，不再输出 deferred 启动摘要
  - 当真正已配置导出目标（`file` / `otlp-http` / `otlp-grpc`）时，仍保留初始化摘要日志

- `src/core/startup-log.ts`
  - 新增启动摘要判定与文案格式化辅助函数

### 测试补充

- `test/startup-log.test.ts`
  - 覆盖 deferred export 默认静默
  - 覆盖 file / otlp-grpc 启动摘要仍保留

### 文档与版本同步

- `README.md`
  - 更新导出日志策略说明
- `CHANGELOG.md`
  - 新增 `1.0.5` 发布记录
- `package.json` / `package-lock.json`
  - 版本升级到 `1.0.5`
